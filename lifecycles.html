React State & Lifecyles

State and lifecycles are important concepts in the React library.

This page will breifly explain the concepts and have an interactive demo along with a code snipet for them.

In React, we have components which can be anything like a textbox, button or a graph visual.
Components can be stateful by having a JavaScript object called state.
This state object stores dynamic data and determines the behavior of the component.
State is private to the component and what that means is that it can only be controlled by
the component itself. Another important detail about state is that it is immutable.

State Code Example 1

        
            
          class Counter extends React.Component {
            constructor(props) {
                super(props);
                this.state = {
                    counter: 0 
                }
                this.increment = () => this.setState({counter: this.state.counter + 1});
                this.decrement = () => this.setState({counter: this.state.counter - 1});
            }

            render() {
                console.log('Render');
                return (
                    <div>
                        <button onClick={this.increment}>Increment</button>
                        <button onClick={this.decrement}>Decrement</button>
                        <div className="counter">
                            Counter: {this.state.counter}
                        </div>
                    </div>
                )
            }
          }

Example 1 is a simple Counter component which has two buttons that increment or decrement the counter value.
We modify that value using state. We initialize the state in the constructor as well has define the methods increment
and decrement. The methods will get the current state, add/subtract 1 then setState to set the new state.
Notice how we can't directly change the state by doing this.state.counter = this.state.counter - 1.
That is because state is an immutable object.
